Module-2

NETWORK SECURITY JUNE 2022 B

Module Home

In this module we will look at the impact of cyber security on network systems and designs, focusing especially on network vulnerability assessments. We will discuss techniques that can be used to scan, evaluate and in some case mitigate network vulnerabilities. It also will consider the area of vulnerability assessments and explain the various mechanisms and approaches encapsulated in the Vulnerability Assessment Continuum.

Unit 1: History of Network Security, Vulnerabilities and Approaches

This week we will review the history of network security and vulnerabilities and relate it to the modern landscape. We will also discuss some security best practices and review the Cyber Kill Chain model proposed by Lockheed Martin (Hutchins et al, 2011), which was based on a study of contemporary breach approaches.

Knowledge:
1. Review the history of network security and vulnerabilities.
2. Discuss a best practice approach to security.
3. Review a number of modern scanning and attack approaches.

Learning Outcome:
1. Explain the origins of a number of common attacks.
2. Recommend a best practice approach to mitigate attacks.
3. Discuss a number of modern protection techniques.

Unit-2: Advanced Persistent Threats: Applying the Cyber Kill Chain Model to a Case Study

Revisiting the Cyber Kill Chain model proposed by Lockheed Martin (Hutchins et al, 2011) and apply it to a modern case study - the solar winds exploit (Temple-Raston, 2021) to explore if the well-known Cyber Kill Chain model is still as applicable for the analysis of modern Advanced Persistent Threats (APTs).

Knowledge:
1. Discuss the impact of APTs and review the Cyber Kill Chain model.
2. Review the solar winds exploit (Temple-Raston, 2021).
3. Evaluate if the model helps explain and mitigate the exploit.

Learning Outcome:
1. Describe the Cyber Kill Chain model.
2. Apply the model to analyse a well-known APT.
3. Describe possible mitigations to avoid a similar exploit.

Unit-3: Vulnerability Assessments

Reviewing the different approaches to vulnerability assessments from paper-based processes such as Cyber Essentials, through a variety of vulnerability assessment approaches, to Crest-based and certified Penetration Tests. The lecturecast focuses on a general vulnerability assessment approach that can be used to create the basic audit document. This document should be used as the basis for more in-depth penetration tests.

Knowledge:
1. Discuss and review the different approaches to vulnerability testing/ assessments.
2. Evaluate and select appropriate testing and scanning tools to perform an assessment.
3. Analyse results and produce a vulnerability assessment report.

Learning Outcome:
1. Select and explain which assessment regime to use based on the type and size of the target business.
2. Utilise open-source scanning and testing tools to perform vulnerability tests and scans.
3. Analyse scanning results and make recommendations to mitigate vulnerabilities.

Unit-4: Breach Analysis and Mitigation

Use the techniques covered so far to review and evaluate security breaches and formulate responses to typical attacks. We will also review the suite of tools contained in the Kali Linux distribution that are focused on security assessments and testing.

Knowledge:
1. Review a number of typical security breaches.
2. Discuss typical mitigations to stated attacks.
3. Evaluate some of the tools include in the Kali Linux software distribution.

Learning Outcome:
1. Describe a typical breach attack.
2. Suggest suitable mitigations against typical attacks.
3. Evaluate and select appropriate tools from the Kali Linux distribution.

Unit-5: Logging, Forensics and Future Trends

This week we will discuss two topics key to breach investigation and resolution: logging and forensics. We will look at the differences between logging on Windows and Linux systems and review key tools used in logging. We will also review the types of investigation that can be carried out using common forensic tools.

Knowledge:
1. Review logging tools and techniques on Windows and Linux.
2. Discuss best practice for the use of logging on both systems.
3. Investigate the use of forensic tools and the data that can be extracted.

Learning Outcome:
1. Specify a logging configuration for Windows/Linux.
2. Analyse the results of log and forensic analysis tools.
3. Select and utilise forensic tools.

Unit-6: The Great Debate: The Future of the Internet

This week aims to bring together the various threads and lessons discussed in the module to answer the question: what is the future of the internet? In answering this question, you should consider technical and security considerations as well as social, legal and privacy challenges.

Knowledge:
1. Review how technological advances are trying to address the shortcomings of the current internet.
2. Discuss a number of alternative approaches to provide a next generation internet.
3. Investigate how these solutions mitigate technical, security, social, legal and privacy challenges.

Learning Outcome:
1. Describe how a number of alternative next generation internet solutions will operate.
2. Debate which potential should (or could) become the next generation internet.
3. Explain how these solutions address key challenges.
