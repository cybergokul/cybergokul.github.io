Module-4

SECURE SOFTWARE DEVELOPMENT NOVEMBER 2022

Module Home

In this module, we’ll take an in depth look at the security risks which are associated with programming languages, from the perspectives of design and architecture approaches, programming paradigms, testing, the impact and implications of operating systems and libraries on software development, the security risks associated with distributed systems and APIs, and future trends in secure software and systems design. Running through these topics will be the concept of the secure software development life cycle.

Unit 1: Introduction to Secure Software Development

The content we will examine this week will underpin a number of the important concepts that will be used throughout the module. This begins with an examination of the management approaches to supporting software development, in recognition of the weaknesses of the more traditional approaches such as waterfall, and the strategies which are applicable to the more modern approaches such as agile. The Unified Modelling Language (UML) will be investigated in Week 1, with knowledge gained being applicable throughout the module in relation to the modelling of software deployments from their design phase.

Knowledge:
1. Explore the waterfall and agile approaches to software development, with a focus on the implications of developing secure software using each.
2. Become acquainted with the Unified Modelling Language and how it can be used to support software development.
3. Gain a familiarity with the standards which are used by industry to develop secure software.
4. Appreciate the importance of developing a risk-aware culture within an organisation.

Learning Outcome:
1. Identify relevant academic literature which discusses the ways in which secure software may be developed using an agile process.
2. Decompose an activity into a set of tasks which can be represented using a flow chart from the UML portfolio of modelling techniques.
3. Describe the importance of creating a culture of risk awareness in an organisation, in addition to the creation of secure software.

Unit 2: UML Modelling to Support Secure System Planning

Welcome to Week 2, during which we will gain some practical skills in the creation of UML flow charts. Flow charts are used during the design stage of the SDLC and can be used from that point onwards to support communications between stakeholders and to act as documentation of the final product created.

Knowledge:
1. Gain an awareness of the opportunities for integrating software security development practices at each stage of the agile SDLC.
2. Understand the range of ways in which software security can be problematic.

Learning Outcome:
1. Research the academic literature (e.g., published by IEEE, ACM, Elsevier or ScienceDirect) on software development which follows an agile process and prioritises security.
2. Become familiar with the language and concepts of a security standard which has been published by the ISO/IEC.

Unit 3: Introduction to Programming Languages

Welcome to Week 3 where we will spend some time investigating programming languages – looking at the history, concepts and design that led to the languages we have today. Programming languages are a fundamental concept in computer science – they are the primary tool used by computer scientists and developers to design, define, create and implement the tools we use today.

Knowledge:
1. Review the history and concepts of programming languages.
2. Investigate language concepts using Python as an example.
3. Describe best practices and methods to overcome common security issues.

Learning Outcome:
1. Describe some key milestones in the development of programming languages.
2. Outline some of the key paradigms that define the different types of languages.
3. Explain the key concepts that determine the operation of programming languages.
4. Discuss key programming challenges and recommended best practices.

Unit 4: Introduction to Testing

Welcome to Week 4. This week, we will explore the art of testing software, both for general quality and, specific to this module, for security. This takes into account the key terms within the context of software testing, in addition to the important software testing techniques. These are placed in context with a consideration of the practices which are used in industry and are reinforced from a practical perspective through harnessing the tools that are available in Python to automate the testing process.

Knowledge:
1. Explore the range of questions to ask when developing a test plan.
2. Become familiar with two industry software testing standards.
3. Examine a range of testing techniques and appreciate the relevance of each in different testing scenarios.
4. Recognise the tools and frameworks which are available to support and automate the Python testing process.
5. Reflect on the utility of an ontology to support a distributed architecture.

Learning Outcome:
1. Describe the key terms which are associated with testing software for security.
2. Prepare a list of questions to ask when designing a test plan for secure software.
3. Design software tests through understanding the range of ways which the security of software can be breached.
4. Design an ontology which can be used in standardised service deployments.

Unit 5: Future trends in Secure Software Development

Welcome to Week 5, during which we will look at a number of trends and advances in both academic research and industry that serve as indicators around the general direction of research and development in the area of secure software development.

Knowledge:
1. Discuss future trends in the areas of secure design, development, programming languages and operating systems.
2. Explain what is meant by Fog Computing, the Internet of Things and Cyber Physical Systems.
3. Explore systems and solutions that can be investigated as possible research topics or fields.

Learning Outcome:
1. Give examples of fog computing, IoT and Cyber Physical System components and solutions.
2. Describe some of the security issues with the above-mentioned systems.
3. Recommend emerging technologies and solutions to investigate.

Unit 6: The Security of Programming Languages Debat

Welcome to Week 6, during which we will consider the security properties of programming languages in use today. This will take into account both the strengths and weaknesses of programming languages from the security perspective, in addition to consideration of the ways in which they may need to be extended.

Knowledge:
1. Review the security capabilities of a selection of programming languages, which include F#, Python, Rust, and Swift.
2. Engage in a debate about the above languages.
3. Explore some of the advances needed to support secure programming.

Learning Outcome:
1. Recommend which programming language is the optimum to adopt to support a secure system.
2. Provide evidence to support your recommendation.
3. Make recommendations on the way(s) in which programming languages might be extended to meet the security needs of our modern world.
